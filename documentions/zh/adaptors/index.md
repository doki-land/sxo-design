# 框架适配器 (Adaptors)

SXO 设计系统提供两种运行模式：**JIT (Just-in-Time) 模式** 和 **AOT (Ahead-of-Time) 模式**，以适应不同的开发场景。

---

## 🔥 JIT (Just-in-Time) 模式
适用于现代前端框架。这种模式分为 **“运行时 JIT”** 和 **“编译期优化 JIT”**（推荐）：

- **编译期优化 (使用 Vite Plugin)**: 通过插件在编辑期收集类名并生成静态 CSS。**实现零运行时开销**，生产环境性能最佳。
- **纯运行时 (无插件)**: 适配器在浏览器端实时调用引擎生成样式。适用于动态性极强或无法使用构建工具的现代框架场景。

### 框架适配器
- [**React**](./react.md) - 提供 Hooks 与 Provider 的深度集成。
- [**Vue 3**](./vue.md) - 基于 Composition API 的高性能适配。
- [**Vue 2**](./vue2.md) - 为存量 Vue 2 项目提供的兼容支持。
- [**SolidJS**](./solid.md) - 利用细粒度响应式实现的极速渲染。
- [**Svelte**](./svelte.md) - 编译型适配，零运行时开销。
- [**Alpine.js**](./alpine.md) - 针对轻量级交互的声明式增强。

### 构建工具
- [**Vite Plugin**](./vite-plugin.md) - 实现自动按需导入与主题热更新的核心工具。

---

## ❄️ AOT (Ahead-of-Time) 模式
适用于静态站点生成器 (SSG) 或传统后端开发框架。通过引入预编译好的 CSS Bundle，实现“零配置、零构建”的集成体验。

### 静态框架 (SSG)
- [**Hexo**](./hexo.md) - 静态博客与文档站点的最佳选择。
- [**Hugo**](./hugo.md) - 极致性能的静态站点集成方案。

### 后端开发框架
- [**Spring Boot**](./spring-boot.md) - 适用于 Thymeleaf、JSP 等后端渲染环境。

### 通用方案
- 如果您的框架未在上方列出，可直接参考 [CDN 快速开始](../../../sxo-cdn.md#🛠️-快速开始示例-html) 进行通用集成。

### 💡 如何选择模式？

| 维度 | JIT 模式 | AOT 模式 |
| :--- | :--- | :--- |
| **适用场景** | 单页应用 (SPA)、复杂交互项目 | 博客、企业官网、传统后台 |
| **构建依赖** | 需要 Vite / Webpack | **零构建依赖**，直接引入 |
| **定制能力** | 极高（运行时动态计算） | 高（通过 CSS 变量覆盖） |
| **性能表现** | CSS 体积最小（按需生成） | 首屏渲染最快（预编译静态资源） |
